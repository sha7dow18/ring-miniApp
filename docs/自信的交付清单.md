## 🏗 架构与工程质量

### 双端切换
- 单小程序方案：`user_profile.role ∈ {elder, child}` 决定 tab-bar 与首页
- 首次启动若 role 为空 → 引导到 `pages/role-switch` 二选一
- profile 页"切换角色"可随时换端，评委一键体验两端

### 家庭绑定
- 老人生成 6 位邀请码 → 子女输入兑换 → 双方 `user_profile.boundFamilyId` 指向同一 `family_bindings` 文档
- CUSTOM 安全规则允许任意登录用户按 inviteCode 查询和更新 binding 记录
- `family_inbox` 仅收件人可读，任意用户可 create — 通道安全合规

### 内容真实度
- **CloudBase Agent "Aita"** 是真实部署的 `ibot-aita` CloudRun function，tools 真实读云数据库
- 体质辨识 / 周简报 / AIGC 内容生成全部调 **真实 DeepSeek + 混元** API（wx.cloud.extend.AI）
- 所有关键业务写入真实云集合，非 localStorage

### 测试
- **304 个单测，30 个 suite，全部通过**
- 覆盖率：stmts **75.58%** / branches **63.51%** / funcs **84.46%** / lines **76.39%**
- 所有 service 的核心纯函数接近 100% 覆盖；云路径有端到端 mock 测试

### 权限模型
```
PRIVATE:  user_profile / health_records / cart_items / orders / chat_sessions / constitution_assessments / replenishment_plans / subscriptions
READONLY: products / content_feed（所有登录用户可读，创建者可写）
CUSTOM:   family_bindings（登录可读，任意登录用户可更新，创建者可删）
          family_inbox（仅收件人可读写，任意登录用户可 create）
          weekly_digests（_openid 或 sharedWith 可读）
```
