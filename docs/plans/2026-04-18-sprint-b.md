# Sprint B · 电商数据闭环

## Goal
把商城从纯展示 + 假按钮做成真实可下单的 mini 电商：商品读云 → 加购物车 → 结算 → 模拟支付 → 订单入库 → 订单列表可查。所有环节除最后一步支付外，全部走真实前后端。

## Why
前面几轮 sprint 把 AI 聊天、健康数据、用户画像都做成了数据闭环。商城模块作为项目第二个核心功能（第一是健康），当前还是纯 mock 展示，点"立即购买"只弹 modal。用户希望除真实支付外，整个电商流程前后端完整打通。

## Scope

### In
- `services/productService.js` - 云读商品目录，空则回退 mock
- `services/cartService.js` - 购物车 CRUD + 合计计算
- `services/orderService.js` - 订单 CRUD + 状态机（pending / paid / shipping / done / canceled）
- `pages/cart/` - 新页，列表 + stepper + 合计 + 去结算
- `pages/checkout/` - 新页，地址 + 订单摘要 + 模拟支付
- `pages/mall` - 改读云 + 右上购物车角标
- `pages/mall-detail` - 改读云 + 加购/立即购买
- `pages/orders` - 改读云 + 状态筛选
- `pages/order-detail` - 改读云 + pending 时的支付按钮 + 联系客服入口
- `docs/seed-data/products.json` + README 告诉用户如何导入

### Out（留给后续）
- 真实 WeChat Pay
- 发货状态自动推进
- 地址簿
- UI 精修（独立 sprint）

## Success Criteria
1. 冷启动 mall 页：若 products 集合为空自动回退 mockStore 的 4 个商品；若已导入则读云
2. 加入购物车：角标实时 +1；切到 cart 页能看到；qty 可增减；合计正确
3. 结算：mall-detail"立即购买"或 cart"去结算"都能进 checkout，payload 正确区分单品/购物车
4. 模拟支付：createOrder → payOrder → clearCart → 跳 order-detail
5. 我的订单：看到刚下的单，状态徽章"已付款"；按 tab 筛选工作
6. 订单详情 pending 态显"立即支付"；非 pending 隐藏；永远有"联系客服"入口
7. `npm test` 全绿，覆盖率阈值维持不降
8. `docs/seed-data/products.json` + 导入指引完整

## Data Contracts

### products
```
{ _id, id, name, category, price, image, imageName, desc, detailPitch, tags[], color, onSale, stock, createdAt }
```

### cart_items
```
{ _id, _openid, productId, qty, addedAt, updatedAt }
```
每用户每商品唯一（业务层：先读后合并）

### orders
```
{ _id, _openid, orderNo, items[], total, address, status, createdAt, payTime, updatedAt }
```
status: `pending | paid | shipping | done | canceled`
