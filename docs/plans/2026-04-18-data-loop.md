# Data Loop Sprint — 健康数据上云 + 会话标签 + 用户画像 + 测试

## Goal
把三件事做成闭环：(1) 健康数据每天一条写云库、首页读云库、AI 聊天注入为上下文；(2) 聊天会话自动打标签（舌诊/睡眠/体质/通用）；(3) 用户头像+昵称通过 WeChat 官方 API 采集，持久化到 `user_profile`。同时引入 jest 覆盖 services 层纯逻辑。

## Why
当前健康指标仅存在 mockStore（localStorage），不持久、AI 拿不到、无法展示趋势。用户数据只有 `users`（空壳），个性化无处落地。会话无法按类型筛选，历史抽屉混在一起。

## Scope
### In
- `health_records` 集合：每日一条，字段见下。前端 mock 生成 + 云端持久。
- `user_profile` 集合：补齐 nickname/avatar/基本资料。
- `chat_sessions.tag`：新字段，首条消息由关键词规则自动打标。
- Home 首页改读云端今日数据 + 新增 7 日趋势条。
- user-info 页改为 `<button open-type="chooseAvatar">` + `<input type="nickname">`，写云。
- AI 聊天：发送前拉最近 3 天健康记录注入 system prompt。
- Jest + 纯逻辑单测：metrics 生成器、会话 tagger、aiService 的 toApiMessages/hasImage/健康上下文格式化。

### Out
- 真实 BLE（设备数据继续 mock 生成）
- 真实支付（商城继续 mock）
- 云函数（全部前端直连 `wx.cloud.database()` + `wx.cloud.extend.AI`）
- `miniprogram-simulate` 组件级测试（wx mocking 成本太高，本轮只测纯逻辑）
- 新增独立 history 页（7 日趋势放 home 上）

## Success Criteria
1. 冷启动后，云库 `health_records` 存在当日记录（若不存在则自动生成并写入）。
2. Home 页"今日指标"数据源是云库，不是 mockStore；刷新按钮写入新记录。
3. Home 页展示近 7 日睡眠评分/HRV/步数 mini 趋势。
4. AI 聊天第一轮响应中能引用当日数据（通过 system prompt 注入验证）。
5. 聊天侧边栏每个会话显示 tag 徽章，首条消息"拍舌头…" → 舌诊，"睡不好" → 睡眠，"什么体质" → 体质，其余 → 通用。
6. user-info 页使用官方 `chooseAvatar` + `nickname` 组件；保存后云库 `user_profile` 有相应记录，`profile` 页展示正确。
7. `jest` 全部通过，覆盖率 ≥ 80% 对受测模块。
8. 数据隔离：以不同 openid 登录无法看到对方数据（依赖集合权限"仅创建者可读写"）。

## Data Contracts

### `health_records` (每日一条)
```
{
  _openid,          // 自动注入
  date: "YYYY-MM-DD",
  sleep_score: 70-97,
  sleep_duration: 420-510,  // min
  deep_sleep_min, rem_min,
  hr_resting: 60-80, hr_max: 120-160,
  hrv: 30-72,
  steps: 4000-12000, calories: 140-600,
  spo2: 95-99,
  stress: 20-70,
  skin_temp_delta: -0.6 ~ +0.6,
  respiratory_rate: 12-18,
  readiness_score: 55-95,
  createdAt, updatedAt
}
```

### `user_profile` (每用户一条)
```
{
  _openid, nickname, avatarUrl,   // avatarUrl 是 cloud fileID
  gender: "男|女|保密", birthday, heightCm, weightKg, phone,
  allergyHistory, medicalHistory,
  createdAt, updatedAt
}
```

### `chat_sessions.tag`
`"舌诊" | "睡眠" | "体质" | "通用"` — 由 `tagFromText(firstText, hasImage)` 纯函数决定。

## Tasks

### T1. 健康服务 + 云持久
- `services/healthService.js`
  - `generateDailyMock(dateStr)` — pure，返回完整健康记录对象
  - `ensureTodayRecord()` — 云查询：若今日无记录则生成并 insert
  - `getTodayRecord()` — 云查询当日最新
  - `getRecent(nDays)` — 云查询近 N 天（倒序）
  - `refreshToday()` — 强制重新 mock 当日并 update
  - `buildAiContext(records)` — pure，把记录数组拼成中文摘要给 AI

### T2. 会话服务 + 打标
- `services/sessionService.js`
  - `tagFromText(text, hasImage)` — pure
  - `createSession(firstText, hasImage)` — 复用 ai-chat 内逻辑，带 tag
  - `updateMessages`, `listSessions(limit=20)`, `loadSession(id)`
- ai-chat 移除原内联 DB helpers，改 require sessionService

### T3. 用户画像服务
- `services/profileService.js`
  - `ensureProfile()` — 首次创建空记录
  - `getProfile()`, `updateProfile(patch)`
  - `uploadAvatar(tempPath)` — 走 `wx.cloud.uploadFile` → `profile-avatars/`

### T4. app.js wire-up
- onLaunch 后 `ensureTodayRecord()` + `ensureProfile()`（静默失败不阻塞）

### T5. aiService 注入上下文
- `sendMessage(history, onChunk, healthContext)` 新增 3 参
- 若有，拼入 system prompt 顶部："用户最近 3 天健康参考：…"

### T6. Home 页迁移
- onShow 调 `healthService.getTodayRecord()` 填 UI
- 下拉刷新 → `refreshToday()`
- 新增"近 7 天"横向小柱状图（canvas）展示 sleep_score/hrv/steps 其中之一（按 tab 切换或固定展示睡眠）

### T7. user-info 页面改造
- WXML：`<button open-type="chooseAvatar" bindchooseavatar="onChooseAvatar">` + `<input type="nickname" bindchange="onNickname">`
- 保存时 uploadAvatar → 存 fileID → updateProfile
- 读写对象：云 `user_profile`，本地不再用 mockProfileService 这部分

### T8. profile 页
- onShow 读 `user_profile`，fallback 到 mockStore（避免空白）

### T9. ai-chat 接入
- createSession 带 tag
- history drawer 每项显示 tag 徽章
- doSend 前调 `healthService.getRecent(3)` → `buildAiContext` → 传给 aiService

### T10. 测试栈
- `package.json` 添加 jest
- `tests/healthGenerator.test.js`，`tests/sessionTagger.test.js`，`tests/aiContext.test.js`，`tests/aiMessages.test.js`
- 纯函数导出，无 wx 依赖
