# UI Polish · 统一养生色设计系统

## Goal
前端经过多轮 sprint 叠加，14 个页面的 wxss 各写各的（3 种绿、2 套背景、5 种圆角、.card/.btn/.row 重复定义 5-6 遍）。本 sprint 建立一套 token + 公共类，迁移所有页面到统一设计语言。**保留现有养生土色/森林绿品牌**，不换配色。

## Why
审计综合评分 3/10。主要问题是样式无来源、无复用、无约束——加页面越多越乱。研究 agent 给的设计系统方向与养生调性不符（医疗青），但其 token 化方法论直接适用。

## Scope

### In
- `miniprogram/styles/tokens.wxss` - CSS 变量（色 / 字号 / 间距 / 圆角 / 阴影 / 字重）
- `miniprogram/styles/common.wxss` - 通用类（.card / .btn-* / .row / .pill / .hover / .safe-bottom / .h1-.caption / .txt-1-3）
- `app.wxss` 引入上面 + `page` 全局默认
- 迁移 14 个页面的 wxss：消除硬编码色值、统一圆角/字号/间距/阴影、删除与公共类等价的本地定义
- `scripts/check-tokens.sh` 残留扫描
- `docs/design-system.md` 设计系统参考

### Out（下一轮）
- `<view bindtap>` → `<button>` / `<navigator>` 语义化迁移
- emoji 图标 → SVG
- WeUI 引入
- skeleton 骨架屏 / 空态插画
- 深色模式

## Success Criteria
1. `miniprogram/styles/tokens.wxss` 和 `common.wxss` 存在，被 app.wxss 引入
2. 所有页面背景统一为 `var(--bg-page) = #F4F0EB`（页面级无渐变）
3. 主色仅 `var(--brand) = #3F6B5E` 和 `var(--brand-dark) = #2A4A3E`（仅作渐变 stop）
4. 所有卡片圆角统一 `var(--r-card) = 24rpx`
5. `.card` 样式定义只在 `common.wxss` 出现一次，各页面 wxss 中等价的定义已删除
6. `scripts/check-tokens.sh` 扫描残留硬编码色值 ≤ 豁免清单长度
7. `npm test` 全绿（116 passing，覆盖率不降）
8. 14 页在微信开发者工具里肉眼看一遍视觉一致（由用户验收）

4. 视觉特色（如血压图颜色、订单状态色）按豁免清单保留
