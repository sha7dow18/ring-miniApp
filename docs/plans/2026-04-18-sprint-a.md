# Sprint A · 清理 + 流式蓝牙 + 客服页 + 修断按钮 + 重置应用

## Goal
把前端从"有些按钮是假的"状态清到"所有已有按钮都真的能用且跳到正确目的地"；同时把死板的健康数据做成持续跳动的流式 mock，视觉上像真蓝牙设备在采样；引入中心化配置文件消除散落的魔法数字。

## Why
前端界面审计（见主对话）发现多处问题：AI 页面 3 选 1 时代结束（只保留 ai-chat），但 ai-lab/ai-records 残留；home 的"去 AI 分析"按钮 switchTab 到非 tabBar 页面导致无效；service 的"设备详情"跳错目标；"退出登录"对微信小程序语义不成立；健康数据是静态一次性生成，没有"设备在跑"的感觉。用户希望所有能按的按钮都真的能按。

## Scope

### In
- 新增 `miniprogram/config/index.js` 集中配置
- 删除 `pages/ai-lab/` 和 `pages/ai-records/`
- 新增 `utils/mockBleStream.js` 流式蓝牙模拟
- 新增 `pages/customer-service/` 客服静态页
- 修 home/service 两处跳转断点
- settings 页"退出登录"→"重置应用"
- jest 覆盖 BLE 纯逻辑

### Out（→ Sprint B）
- 商品/购物车/订单（products、cart_items、orders 集合虽已建好但不动）
- 富消息 card 类型扩展
- 真实 WeChat Pay 集成

## Success Criteria

1. `miniprogram/config/index.js` 存在，`app.js`/`aiService.js` 所有环境/模型名/时间间隔从 config 读，无散落魔法数字
2. app.json pages 数组不含 ai-lab / ai-records；两目录删除；profile 菜单无"AI 分析记录"项
3. 小程序启动后 home 页心率/步数/HRV/SpO2 数字每 3 秒跳一下
4. 5 分钟后观察云库 health_records 当日记录的 updatedAt 变化，steps 累加
5. home 页"去 AI 分析"能正确切到 AI tab
6. service 页"设备详情"能打开 device-detail（显示 MAC、固件版本等硬件信息）
7. mall-detail "联系客服"能进 customer-service 页，显示客服微信/邮箱/服务时间
8. settings 页有"重置应用"按钮，点击二次确认后执行 `wx.clearStorageSync()` + mockStore 重置
9. `npm test` 全绿；覆盖率门槛 stmts≥70/funcs≥75/lines≥70/branches≥55 保持不破

## Tasks

### T1 · 配置文件
- `miniprogram/config/index.js` 导出：
  - `cloud.env`
  - `ble.tickInterval`（ms）、`ble.aggregateInterval`（ms）、`ble.seed`（各指标初始值）、`ble.walk`（各指标单 tick 波动范围）
  - `ai.textProvider/textModel/visionProvider/visionModel`
  - `app.versionName`

### T2 · 删冗余 AI 页面
- rm `pages/ai-lab/*`、`pages/ai-records/*`
- `app.json` pages 数组去掉这两项
- `pages/profile/{index.js,index.wxml}` 删除"AI 分析记录"菜单和 handler

### T3 · 流式蓝牙
- `utils/mockBleStream.js`：
  - 纯函数：`tick(prev, config)` → next snapshot（心率±1~3、HRV±1~2、步数 +3~20、SpO2±0~1、压力±0~2、体温±0~0.05）
  - 纯函数：`aggregate(snapshots)` → { avg/min/max/sum }
  - impure：`start/stop/subscribe/unsubscribe/getSnapshot`，5 分钟自动 aggregate + update 云 health_records
- `app.js` onLaunch 启动
- `pages/home/index.js` onShow 订阅 → setData 实时数字；onHide unsubscribe
- home 页"今日心率/HRV/SpO2/步数"等字段值改从 stream 读；下拉刷新仍调 `refreshToday`

### T4 · 客服页
- `pages/customer-service/index.{js,wxml,wxss,json}`
  - 展示：客服微信号（长按复制）、邮箱、工作时间、3 条常见问题
  - 注册到 app.json
- `pages/mall-detail/index.js` `onContact` → `wx.navigateTo("/pages/customer-service/index")`

### T5 · 修跳转
- `pages/home/index.js` `goToAi` → `wx.switchTab({ url: "/pages/ai-chat/index" })`
- `pages/service/index.js` `onViewDeviceDetail` → `wx.navigateTo({ url: "/pages/device-detail/index" })`

### T6 · 重置应用
- `pages/settings/`：按钮文案改"重置应用"
- handler：modal 二次确认 → `wx.clearStorageSync()` + `mockStore.hydrate()` + toast

### T7 · 测试
- `tests/mockBleStream.test.js`：
  - tick 返回快照字段齐全
  - 波动幅度在配置范围内
  - 100 次 tick 后值不越过物理边界
  - aggregate 均值/最大/最小/sum 正确

### T8 · 文档
- 本 plan
- `docs/README.md` 加 BLE Stream 章节
