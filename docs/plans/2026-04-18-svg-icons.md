# emoji → SVG 图标迁移 · 重档

## Goal
移除所有 emoji 作为 UI 图标的用法，替换为统一的 SVG 图标集（lucide 风格）。空态单独做养生调性插画。AI tabBar 的 AI 占位图标换成听诊器，更贴合"健康助手"语义。

## Why
emoji 跨设备渲染不一致（iOS 拟物 vs Android 扁平），色彩与养生主题冲突，字号控制差，无障碍差。研究 agent 和业界 pattern 共识是"UI 图标不要用 emoji"。

## Scope

### In
- 14 个基础图标 (lucide) 烘焙 brand 色到 `assets/icons/`
- 1 个反色变体 `assets/icons/inv/message-circle.svg`（customer-service hero 深色底）
- 5 个订单状态图标反色 `assets/icons/status/*.svg`（hero 有渐变底，用白色）
- 2 个空态插画 `assets/icons/empty/cart.svg` / `orders.svg` — 养生调性，≥3 元素
- AI tabBar 听诊器 SVG 替换 `ai.png` / `ai_active.png`
- home / ai-chat / customer-service / checkout / mall / cart / orders / order-detail 8 页 WXML + JS 改动
- `<image>` 元素加 `aria-label` 中文
- `scripts/check-tokens.sh` 扩展扫描 emoji
- `docs/design-system.md` 增加"图标"章节

### Out
- iconfont.cn 字体图标
- 自定义 `<svg-icon>` 组件（直接 `<image src>` 足够）
- 动画图标
