# 微信小程序后端架构选型

> 总结：微信小程序能挂什么后端？免费额度多少？备案要不要？什么时候该换方案？

## TL;DR

三条主流路线：

| 路线 | 备案 | 域名 | 运维 | 上限 | 入门成本 |
|---|---|---|---|---|---|
| **微信云开发** | ❌ 不用 | ❌ 不用 | 0 | 受套餐限制 | 5 分钟 |
| **自建服务器** | ✅ 必须 | ✅ 必须 | 高 | 无限 | 几天 |
| **腾讯云/阿里云 BaaS** | ✅ 必须 | ✅ 必须 | 中 | 无限 | 1-2 天 |

**原型阶段闭眼选微信云开发**。有真实用户后再视情况迁移。

## 路线 1：微信云开发（CloudBase）

### 本质

腾讯云 CloudBase 的一个"面向微信小程序的特供版"。本质是 serverless，包含：

- **云函数**（Cloud Functions）— Node.js 16/18/20 运行时
- **云数据库**（MongoDB 协议）
- **云存储**（对象存储）
- **云调用**（调微信 API 不用 access_token）
- **HTTP 访问服务** — 把云函数暴露成 URL
- **AI 扩展** — `wx.cloud.extend.AI` 内置 AI 能力

### 最大卖点

**不用域名、不用备案**。小程序和云开发之间有内置信任链，`wx.cloud.callFunction()` 直接调，不走 HTTP。

对学生、个人开发者来说，省掉备案就省掉 20+ 个工作日的等待和一堆资料。

### 免费额度

2025 年 3 月 13 日起，微信云开发推出新的免费政策：

- **免费云环境**：每个小程序账号可以创建 **1 个免费云环境**
- **开发阶段免费**：免费环境在小程序**正式发布上线前**可以无限期使用
- **上线后**：必须购买基础套餐，否则资源受限
- **AI Token**：免费环境内含少量 token，不能加购，要更多需升级套餐

### 基础套餐价格（参考，以官方为准）

- 最便宜的付费套餐约 **¥19.9/月** 起
- 按量付费项目包括：云函数调用次数、数据库读写、存储空间、AI token、HTTP 流量

### 适用场景

✅ 学生 / 个人开发者 / 初创团队做原型
✅ 只给微信小程序用的后端
✅ 需要快速验证产品
✅ 不想碰域名备案

### 不适用

❌ 需要同时服务 Web / iOS / Android 等多端
❌ 用户量极大（几十万 DAU+），按量付费会比自建贵
❌ 需要海外部署或合规（云开发只在大陆）
❌ 有复杂的 DevOps 需求（CI/CD、灰度、多环境）

### 最小开通流程

1. 真实 AppID（不是测试号）
2. 微信开发者工具打开项目
3. 顶部工具栏点「云开发」按钮
4. 第一次会引导创建环境，取个名字（比如 `dev`）
5. 记下环境 ID
6. `app.js` 的 `onLaunch` 里调用 `wx.cloud.init({ env: "环境ID" })`

之后就可以 `wx.cloud.callFunction`、`wx.cloud.database()`、`wx.cloud.uploadFile()`。

## 路线 2：自建服务器

### 架构

```
小程序 ──HTTPS──> 你的服务器 (Nginx + Node/Python/Go)
                    │
                    ├──> 数据库 (MySQL/Postgres/MongoDB)
                    ├──> 对象存储 (S3/OSS)
                    └──> 大模型 API
```

### 硬性要求

这是坑最多的路线：

1. **域名**：必须有独立域名
2. **ICP 备案**：域名必须完成大陆 ICP 备案（个人备案约 20 工作日）
3. **HTTPS**：必须 TLS 1.2+，自签证书不行
4. **白名单**：域名要在小程序管理后台「开发 → 开发管理 → 开发设置 → 服务器域名」里配置
5. **对应的 API 类型**：`request 合法域名`、`socket 合法域名`、`uploadFile 合法域名`、`downloadFile 合法域名` 分开配置

### 免费额度

看你挂哪家云：

- **腾讯云/阿里云轻量服务器**：约 ¥24-50/月 起（2 核 2G）
- **Vercel / Cloudflare Pages**：前端托管免费，但小程序用不了前端
- **Cloudflare Workers**：Serverless 函数免费额度可观，但要**国际访问**，小程序在大陆用需要曲线操作

### 适用场景

✅ 已经有成熟的后端服务，小程序只是又一个客户端
✅ 需要多端共用（Web + 小程序 + App）
✅ 用户量大，按量付费不划算
✅ 有 DevOps 团队或有能力折腾

### 完全不适用

❌ 没备案（小程序调不通）
❌ 想用 OpenAI/Claude 等境外 API（服务器必须能访问境外 + 有合规策略）

### 小程序特有的坑

1. `wx.request` 不支持 `302` 重定向，后端要自己处理
2. Cookies 在小程序里处理很麻烦，建议用 token-based 鉴权
3. WebSocket 的 `wx.connectSocket` 和浏览器 `WebSocket` 略有不同，注意心跳策略
4. `wx.uploadFile` 和 `wx.request` 的 header/body 处理不一样，踩过的人都懂

## 路线 3：腾讯云/阿里云 BaaS

这是介于「云开发」和「自建」之间的中间选择：用云厂商的 Serverless 产品（SCF / FC）+ 数据库 + 对象存储拼起来。

### 特点

- 比云开发自由（能配置 VPC、用自定义域名、接更多服务）
- 比自建省心（不用管服务器运维）
- 但仍然需要域名和备案
- 价格介于两者之间

### 适用场景

✅ 需要比云开发更灵活的配置（如自定义 VPC、跨区部署）
✅ 想用腾讯云/阿里云的其他产品（CDN、短信、人脸识别、百炼 AI 等）
✅ 已经在这些云上有其他项目

## 迁移策略

原型 → 上线 → 规模化的典型演进路径：

```
阶段 1: 原型
  微信云开发（免费环境）
     ↓
阶段 2: 灰度上线 (< 1000 DAU)
  微信云开发（付费基础套餐 ¥20-100/月）
     ↓
阶段 3: 小规模商用 (1000-10000 DAU)
  微信云开发（按量付费）或 腾讯云 SCF + CloudBase
     ↓
阶段 4: 大规模
  自建服务器 (K8s / ECS 集群)
  或 云厂商 BaaS + CDN + 数据库独立
```

**关键设计原则**：
- 前端调用接口**封装成 service 层**（而不是直接散落在页面里写 `wx.cloud.callFunction`）
- service 层签名稳定，内部实现可以从 mock → 云函数 → 自建 API 逐步替换
- 数据库表结构尽量不依赖 MongoDB 特有能力（方便未来切到关系型）

## 云开发 vs 自建 的决策清单

选云开发如果：

- [ ] 只给微信小程序用
- [ ] 团队规模 1-5 人
- [ ] 没有 DevOps 资源
- [ ] 一年预算 < ¥5000
- [ ] 数据量 < 10GB
- [ ] QPS < 100
- [ ] 不需要境外 API

任何一条不满足，考虑自建或混合方案。
