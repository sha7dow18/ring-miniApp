## 设计目标
- 让“我的”页面具备科技感、简洁与易用性，统一视觉规范、图标与动效。
- 保持现有功能与入口完整，同时提升信息层级和感知品质。

## 页面结构
- 顶部渐变英雄区（Hero）
  - 内容：头像（圆形，带细描边）、昵称、陪伴时间（如“陪伴您 318 天”）
  - 操作：整块可点击，进入“设置”页
  - 视觉：线性渐变背景 + 半透明玻璃化卡片（glassmorphism），毛玻璃效果不透明度 0.8，柔和阴影
- 我的设备卡片
  - 展示设备名称与连接状态（点位灯：绿色已连接/灰色未连接）
  - 操作：右侧“添加/管理”按钮；点击进入“设备设置”
  - 支持多设备时列表展示（单设备为一行）
- 功能栅格（两列）
  - 条目：问题反馈 / 使用手册 / 常见问题 / 版本更新 / 关于我们 / 数据同步
  - 每项带一致的线性图标与标签文案，卡片式点击态（缩放 0.98 + 阴影增强）
- 底部安全区与留白
  - 预留 16–24px 底部间距，避免与系统手势冲突

## 视觉规范
- 颜色
  - 背景：#0B0F1A → #111827 渐变（深色科技感）或浅色渐变 #F2F5FA → #FFFFFF（可选）
  - 主题：#1AAD19（现有绿）保留为主按钮；辅色 #5B8DEF（科技蓝）用于图标与点位灯
  - 文本：主标题 #FFFFFF、次级 #9CA3AF（深色主题）；浅色主题时主文本 #111827、次级 #6B7280
- 阴影与圆角
  - 卡片圆角 12–16px；阴影 rgba(0,0,0,0.25) y=8 blur=24
- 动效
  - 点击缩放 0.98，过渡 120ms；悬浮（按压）阴影增强；列表项右侧箭头细微位移
- 字体
  - 继承已设置的跨平台回退链（PingFang / YaHei / Hiragino / Segoe / Roboto / Noto 等）

## 组件与实现
- 新增全局样式变量（科技主题）
  - 在 `app.wxss` 或 `pages/profile/index.wxss` 中定义 CSS 变量：`--bg-grad-start`, `--bg-grad-end`, `--card-bg`, `--card-shadow`, `--text-primary`, `--text-secondary`
- 顶部 Hero 区
  - WXML：`<view class="hero"><image class="avatar" .../><view class="hero-info"><text class="name">{{nickname}}</text><text class="days">陪伴您 {{companionshipDays}} 天</text></view></view>`
  - WXSS：渐变背景 + 玻璃化卡片（半透明白 + 背景滤镜）
- 设备卡片
  - 状态点位：`<view class="status-dot connected"></view>`；通过 `connected` 状态切换颜色
  - 管理按钮右对齐；未连接时文案“添加设备”并跳转
- 功能栅格
  - 两列网格：`grid-template-columns: repeat(2, 1fr)`，图标统一尺寸 24px，标签单行省略
- Skeleton 加载与空态
  - 头像/昵称/天数未取到时显示骨架占位；设备为空时展示插画与“添加设备”按钮

## 数据与交互
- 顶部信息
  - 头像、昵称来自 `userProfile`（本地或云端）；陪伴时间来自 `registeredAt`（未设置默认 1 天）
- 跳转
  - 点击 Hero 整块 → `pages/settings/index`
  - 设备卡片右侧按钮 → `pages/settings/device/index`
  - 栅格条目分别跳转既有/新增页面
